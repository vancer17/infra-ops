## 一、结论

**Hub 管理面 Nginx 网关已部署并通过实机验收。**

- `https://10.200.0.1/health` 返回 `ok`（ci-01 与办公笔记本）
- TCP 80/443 在 WG 内可达；HTTP 自动 301 跳转 HTTPS
- `jms.internal` 反代返回 **502**（JumpServer 未部署，**预期**）

---

## 二、部署内容

| vhost | 地址 | 状态 |
|-------|------|------|
| 默认 | `hub.internal`、`10.200.0.1` | `/health` → 200 `ok` |
| JumpServer 预留 | `jms.internal` | 502（上游 `127.0.0.1:8080` 未监听） |

TLS：内网自签（`hub.internal` / `jms.internal` SAN）；Nginx `1.22.1`。

## 三、功能验收

### 3.1 CI 机（ci-01，`10.200.0.2`）

| 检查 | 命令/现象 | 结果 |
|------|-----------|------|
| TCP 443 | `nc -zv 10.200.0.1 443` | succeeded |
| TCP 80 | `nc -zv 10.200.0.1 80` | succeeded |
| HTTPS 健康检查 | `curl -k https://10.200.0.1/health` | `ok` |
| jms vhost | `curl -k --resolve jms.internal:443:10.200.0.1 https://jms.internal/` | HTTP 502（预期） |
| HTTP→HTTPS | `curl -k -I http://10.200.0.1/health` | `301` → `https://10.200.0.1/health` |

### 3.2 办公笔记本（WG Client）

| 检查 | 结果 |
|------|------|
| `ping 10.200.0.1` | 0% 丢包 |
| `nc 10.200.0.1 443` | open |
| `curl -k https://10.200.0.1/health` | `ok` |
