# ECS 企业环境实施方案

基于前述讨论（Dev/Test/Prod 三分环境、WireGuard 零信任内网、JumpServer 堡垒机审计、RDS/OSS、宝塔收敛、Debian 标准化），以下为可直接按阶段执行的 **完整实施方案**。

---

## 一、项目目标与范围

### 1.1 建设目标

| 目标 | 说明 |
|------|------|
| 环境隔离 | 开发、测试、生产独立服务器、独立配置、独立数据 |
| 网络安全 | 业务机不暴露 SSH；运维经 WireGuard + 堡垒机 |
| 操作审计 | 人工 SSH/数据库操作全程留痕（命令 + 录像） |
| 标准化发布 | CI/CD 自动部署，禁止手工改 Prod |
| 云资源规范 | RDS 分库分权，OSS 分前缀/Bucket，RAM 最小权限 |

- 双活/异地灾备（首期做单区域 + 备份演练）
- 等保测评取证（本期按等保思路建设，测评可二期）

## 二、总体架构

```
                         [ 运维 / 开发 ]
                                │
                    WireGuard Client (UDP 51820)
                                │
              ┌─────────────────┴─────────────────┐
              │     Hub: 47.97.19.58               │
              │  WireGuard Server  10.200.0.1      │
              │  JumpServer        10.200.0.1:443  │
              │  Prometheus/Grafana (可选同机)      │
              └─────────────────┬─────────────────┘
                                │  WG 内网 (Hub-Spoke)
        ┌───────────────────────┼───────────────────────┐
        │                       │                       │
   Dev 10.200.1.0/24      Test 10.200.2.0/24     Prod 10.200.3.0/24
   .20 / .68              .105 / .16            .51 / ...
        │                       │                       │
        └───────────────────────┴───────────────────────┘
                                │
                    ┌───────────┴───────────┐
                    │   阿里云 RDS / OSS     │
                    └───────────────────────┘

公网用户 ──HTTPS──→ Prod-01 (120.26.35.51) ──内网──→ Prod 应用集群
```

**三条铁律**

1. 除 Hub（WireGuard）和 Prod 入口（443）外，**ECS 不对公网开放入站**。
2. **人工运维只走 JumpServer**；禁止宝塔终端、本地直连 SSH/RDS。
3. **应用发布只走 CI/CD**；禁止 SCP 手工上 Prod。

## 四、网络与 WireGuard 规划

### 4.1 IP 地址规划

| 网段 | 用途 | 节点示例 |
|------|------|----------|
| `10.200.0.0/24` | 管理与安全 | Hub `.1`，CI `.2` |
| `10.200.1.0/24` | Dev | Dev-01 `.1`，Dev-02 `.2` |
| `10.200.2.0/24` | Test | Test-01 `.1`，Test-02 `.2` |
| `10.200.3.0/24` | Prod | Prod-01 `.1`（入口）～ `.4` |

每台 ECS 的 WireGuard 地址与公网 IP 固定映射，写入资产台账。

### 4.2 Hub-Spoke 拓扑

- **唯一 WG Server**：Hub（47.97.19.58），公网开放 **UDP 51820**。
- 其余 ECS 及运维笔记本均为 **Peer**，`PersistentKeepalive = 25`。
- **CI 机**作为 Peer，部署时通过 WG 内网 SSH 到各环境（不用公网）。

### 4.3 AllowedIPs 与隔离策略

| 主体 | 可路由网段 |
|------|------------|
| 开发人员 WG Client | `10.200.0.0/24`，`10.200.1.0/24`，`10.200.2.0/24` |
| 运维人员 WG Client | `10.200.0.0/16`（全网） |
| Dev ECS Peer | `10.200.0.0/24`，`10.200.1.0/24` |
| Test ECS Peer | `10.200.0.0/24`，`10.200.2.0/24` |
| Prod ECS Peer | `10.200.0.0/24`，`10.200.3.0/24` |
| **禁止** | Dev/Test Peer 路由到 `10.200.3.0/24` |

### 4.4 阿里云安全组规则（模板）

**Hub 安全组**

| 方向 | 端口 | 源 | 说明 |
|------|------|-----|------|
| 入站 | UDP 51820 | 公司出口 IP / 运维 IP 段 | WireGuard |
| 入站 | TCP 443 | `10.200.0.0/16` | JumpServer（仅 WG 网段） |
| 入站 | TCP 22 | 堡垒机自身维护 | 初期可限公司 IP，后期仅 WG |
| 出站 | 全部 | 0.0.0.0/0 | 更新、拉镜像 |

**Dev / Test / CI 安全组**

| 方向 | 端口 | 源 | 说明 |
|------|------|-----|------|
| 入站 | TCP 22 | Hub 内网 IP / `10.200.0.1` | 仅堡垒机跳板 |
| 入站 | 应用端口 | 同环境 WG 网段 | 如 8080 |
| 入站 | 其余 | 拒绝 | 默认拒绝 |

**Prod-01（入口）安全组**

| 方向 | 端口 | 源 | 说明 |
|------|------|-----|------|
| 入站 | TCP 80/443 | 0.0.0.0/0 | 公网 Web |
| 入站 | TCP 22 | Hub / 堡垒机 | 运维 |
| 其余 Prod 节点 | 无公网入站 | — | 仅内网 + WG |

### 4.5 内网 DNS（可选，建议做）

在 Hub 部署 **dnsmasq / CoreDNS**：

| 域名 | 解析 |
|------|------|
| `hub.internal` | 10.200.0.1 |
| `jms.internal` | 10.200.0.1 |
| `ci.internal` | 10.200.0.2 |
| `dev-app.internal` | Dev WG IP |
| `test-app.internal` | Test WG IP |

WG Client 配置 `DNS = 10.200.0.1`。

## 五、三环境业务布局

### 5.1 各环境组件

| 组件 | Dev | Test | Prod |
|------|-----|------|------|
| Web/API | Dev-01、Dev-02 | Test-01 | Prod-02、Prod-03 |
| Nginx 反代 | 可选单机 | Test-01 兼任 | **Prod-01 独占** |
| Worker/队列 | Dev-02 | Test-02 | Prod-04 |
| Redis | Dev-02 本地容器 | Test 容器 | Prod 容器或云 Redis（二期） |
| 数据库 | `app_dev` | `app_test` | `app_prod` |
| OSS 路径 | `dev/` 或独立 Bucket | `test/` | `prod/` |
| 对外域名 | 无公网，仅 `*.internal` | 可选受限演示域名 | 正式域名 + CDN |

### 5.2 应用部署方式（统一标准）

- 运行时：**Docker + Docker Compose**（首期）；每个环境相同 `compose` 结构，不同 `.env`。
- 进程管理：`systemd` 拉起 Compose 或 `docker compose up -d`。
- 配置：环境变量注入，**密钥不进 Git**。
- 日志：容器 stdout → 宿主机目录或 SLS Agent。

### 5.3 域名与证书

| 环境 | 访问方式 | 证书 |
|------|----------|------|
| Dev | WG + `dev-app.internal` | 内网自签或不做 HTTPS |
| Test | WG 或 IP 白名单 + Basic Auth | Let's Encrypt |
| Prod | 公网域名 | 阿里云证书或 Let's Encrypt + CDN |

## 六、数据库方案（RDS）

### 6.1 实例策略

**首期（成本友好）**：同一 RDS 实例，三个库：

- `app_dev`
- `app_test`
- `app_prod`

**推荐（更规范）**：Test/Prod 分实例；Dev 用小规格 RDS 或 Dev 机本地 MySQL（仅开发数据）。

### 6.2 账号与权限

| 账号 | 权限范围 | 使用方 |
|------|----------|--------|
| `app_dev` | 仅 `app_dev` 读写 | Dev 应用 |
| `app_test` | 仅 `app_test` 读写 | Test 应用 |
| `app_prod` | 仅 `app_prod` 读写 | Prod 应用 |
| `ops_readonly` | `app_prod` 只读 | 堡垒机 DBA 角色 |
| `ops_dba` | Prod 读写 | 堡垒机 + 审批 |

**禁止**应用账号拥有跨库权限；**禁止**`root` 给应用使用。

### 6.3 白名单

- 仅各环境 ECS **内网 IP** + 必要时 Hub（数据库代理）。
- **禁止** `0.0.0.0/0`。
- `116.62.228.105`（Test-01）与 RDS 保持 **同可用区**，降低延迟。

### 6.4 备份

- 自动备份保留 **7～30 天**。
- 每月 **恢复演练** 到 Test 库或临时实例。
- 大版本迁移：Test 先执行，Prod 维护窗口执行。

## 七、OSS 对象存储

### 7.1 隔离方式

**方案 A（推荐）**：单 Bucket `yzx-file-storage`，前缀隔离：

```
dev/
test/
prod/
audit/jumpserver/   ← 堡垒机录像归档
```

`taoyouxiangzu` 作为业务子项目，同样 `dev/`、`test/`、`prod/` 前缀。

**方案 B**：每环境独立 Bucket（权限更清晰，成本略高）。

### 7.2 RAM 权限

每台 ECS 绑定 **ECS 实例 RAM 角色**：

| 角色 | OSS 权限 |
|------|----------|
| Dev-ECS-Role | `yzx-file-storage/dev/*` 读写 |
| Test-ECS-Role | `yzx-file-storage/test/*` 读写 |
| Prod-ECS-Role | `yzx-file-storage/prod/*` 读写 |
| Audit-Role | `audit/jumpserver/*` 仅 Put（Hub 录像上传） |

- 关闭 Bucket **公共读**。
- 对外文件用 **签名 URL** 或 CDN 回源鉴权。
- Prod 开启 **版本控制**。

## 八、堡垒机（JumpServer）方案

### 8.1 部署

- **位置**：Hub（47.97.19.58）。
- **方式**：官方 Docker Compose 单机部署。
- **访问**：`https://jms.internal` 或 `https://10.200.0.1`（仅 WG 可达）。
- **HTTPS**：内网 CA 或 Let's Encrypt（若有内网域名解析到 Hub）。

### 8.2 组件

| 组件 | 功能 |
|------|------|
| Core | 用户、权限、审计、Web |
| Lion / Koko | SSH 会话代理、录像 |
| Chen | MySQL 代理 → RDS 人工查库 |

### 8.3 资产分组

| 资产组 | 资产 | 授权 |
|--------|------|------|
| Dev-ECS | Dev 两台 | 开发组 |
| Test-ECS | Test 两台 | 开发组、测试组 |
| Prod-ECS | Prod 四台 | 运维组（可加工单） |
| Prod-RDS | RDS MySQL | DBA；开发仅 `ops_readonly` 只读 |

### 8.4 系统用户（目标机 Linux 用户）

| 用户 | 用途 | 使用环境 |
|------|------|----------|
| `jump_ops` | 日常运维 | Test/Prod |
| `jump_readonly` | 只读排查 | Prod |
| `deploy` | CI 自动部署专用 | 全环境（**不走堡垒机**） |

目标机 SSH：`PasswordAuthentication no`，仅密钥；`deploy` 密钥由 CI 保管，**不录入 JumpServer**。

### 8.5 审计策略

| 项目 | 要求 |
|------|------|
| 登录 | 一人一号 + TOTP MFA |
| 命令记录 | 全量开启 |
| 会话录像 | 全量开启，Prod 保留 ≥1 年 |
| 命令过滤（Prod） | 拦截 `rm -rf`、`mkfs`、`iptables -F`、未授权 `mysqldump` 等 |
| 告警 | Webhook → 钉钉/飞书：非工作时间 Prod 登录、过滤触发、登录失败 |

### 8.6 录像存储

- 热数据：Hub 本地 200G 数据盘，保留 30 天。
- 冷归档：同步至 `yzx-file-storage/audit/jumpserver/`，本地删除旧文件。
- 审计库 PostgreSQL：每日备份。

## 九、CI/CD 方案

### 9.1 部署位置

**47.98.161.33（CI 机）**，WG 地址 `10.200.0.2`。

### 9.2 工具选型（二选一）

| 方案 | 组成 |
|------|------|
| A（常见） | GitLab CE + GitLab Runner + Harbor（私有镜像仓） |
| B | Gitea + Jenkins + Registry |

### 9.3 流水线阶段

```
Push/MR → 单元测试 → 构建镜像 → 推送 Registry
       → 自动部署 Dev
       → 合并 main/release → 自动部署 Test
       → 打 Tag v* + 人工审批 → 部署 Prod（滚动）
```

### 9.4 环境对应分支

| 分支/触发 | 目标环境 |
|-----------|----------|
| `develop` | Dev |
| `main` / `release/*` | Test |
| `tag v*` + 审批 | Prod |

### 9.5 部署执行方式

- CI 通过 WG 内网 SSH 到目标机用户 `deploy`，执行 `docker compose pull && up -d`。
- 或 CI 装 Ansible，Playbook 统一十台配置。
- **数据库迁移**：Flyway/Liquibase，Test 自动跑，Prod 需审批步骤。

### 9.6 与堡垒机边界

| 操作 | 通道 |
|------|------|
| 人工登录、排障、查库 | JumpServer |
| 自动构建、发布 | CI + `deploy` 密钥 |
| 紧急热修 | 仍走 Git + CI 回滚，**禁止**手工改 Prod 代码 |

## 十、监控、日志与告警

### 10.1 监控（Hub 或 CI 机部署）

- **Prometheus**：抓取各节点 `node_exporter`、容器 metrics。
- **Grafana**：按 `env` 标签分 Dev/Test/Prod 看板。
- 告警：磁盘 >85%、CPU 持续高、容器重启、Nginx 5xx 率。

### 10.2 日志

- 应用日志：挂载到 `/var/log/app/`，按天滚动。
- 集中：阿里云 **SLS** 或轻量 **Loki**（Hub）。
- 字段必须含：`env`、`service`、`trace_id`。

### 10.3 告警通道

钉钉/飞书 Webhook，Prod 告警 @ 值班人。

## 十一、安全加固清单（实施必做）

### 11.1 主机

- [ ] SSH 仅密钥，禁用 root 远程登录
- [ ] `fail2ban` 防暴力破解（若有短暂公网 SSH）
- [ ] 自动安全更新（Dev/Test 可开；Prod 维护窗口）
- [ ] NTP 时间同步（审计时间戳准确）

### 11.2 网络

- [ ] 安全组默认拒绝
- [ ] 业务机无公网 SSH
- [ ] WG + 堡垒机双入口，无第三入口（关宝塔终端、关面板公网）

### 11.3 应用与密钥

- [ ] 密钥放阿里云 KMS 或 CI 密文变量
- [ ] 镜像扫描（Trivy）接入流水线
- [ ] Prod HTTPS 强制，HSTS

### 11.4 人员与权限

- [ ] 入职：WG 配置 + 堡垒机账号
- [ ] 离职：当日吊销 WG Peer、禁用堡垒机账号
- [ ] 每季度权限复核

## 十四、运维制度摘要

### 14.1 日常变更

- 配置变更 → Git MR → Review → 合并 → 流水线部署。
- 紧急故障 → 堡垒机排障 → 修复进 Git → CI 发布；**禁止**面板改 Prod。

### 14.2 发布窗口

- Test：工作日自动。
- Prod：周二/周四 20:00～22:00（示例），需审批。

### 14.3 备份

| 对象 | 频率 | 保留 |
|------|------|------|
| RDS | 自动日备 | 30 天 |
| OSS Prod | 版本控制 | 按策略 |
| 堡垒机审计库 | 日备 | 180 天 |
| ECS 快照 | 周备 | 4 周 |

### 14.4 应急

- **应用故障**：Nginx 切流量 / Compose 回滚上一 Tag。
- **数据库误操作**：RDS 按时间点恢复 → 在 Test 验证 → 决策是否覆盖 Prod。
- **堡垒机故障**：临时限源 IP SSH（应急预案，事后补审计说明）；优先恢复 Hub。

## 十五、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| WG Hub 单点 | 全网运维中断 | 配置备份；二期双 Hub |
| 录像盘满 | 无法新会话 | 监控 + OSS 自动归档 |
| 团队习惯宝塔 | 绕过审计 | 制度 + 关面板终端 + 安全组 |
| CI 密钥泄露 | 未授权部署 | `deploy` 限 IP；密钥轮换；仅 WG 可达 |
| RDS 单实例 | Prod/Test 资源争用 | 流量大时 Prod 独立实例 |
| 实施周期拉长 | 环境混乱 | 严格按阶段验收，不跳步 |
