> **前提上下文**：阶段 F（WireGuard）已完成；阶段 G（RDS `app_dev`）已验收；**阶段 H（Dev OSS）已于 2026-06-22 验收**；JumpServer G4/G5 已上线。

## 一、文档用途与维护规则

| 原则 | 说明 |
|------|------|
| 控制台变更须回写 | 修改 Bucket 权限、RAM 策略/角色、ECS 实例 RAM 绑定后，同步更新本文档与 `registry.yaml`、`dev-*.yaml` |
| ECS 走实例 RAM 角色 | Dev **禁止**在应用 `.env` 或 Git 中存放长期 OSS AccessKey |
| 按环境隔离 | **2026-06-22 起**：Dev 使用**独立 Bucket** `infra-dev-file-storage`（资源组 `InfraDevOss`），与生产 `yzx-file-storage` 桶级隔离 |
| 与生产数据隔离须最小权限 | `yzx-file-storage` 已有约 **27 GB** 数据；Dev 角色策略 `DevOssInfraDevFileStorageOnly` **仅允许** Dev 桶，**显式 Deny** 生产桶 |
| 不走 WireGuard | OSS 经阿里云 VPC 内网 Endpoint 或公网 Endpoint 访问，**不经过** `10.200.x.x` WG 隧道 |

**维护流程**：

1. 阿里云 OSS / RAM 控制台确认变更  
2. 更新本文档对应章节  
3. 同步 `docs/assets/registry.yaml`、`docs/assets/dev-01.yaml`（及 dev-02）  
4. 验收通过后更新 `docs/acceptance/20260622-阶段H-Dev-OSS验收.md` 与 [开发环境业务部署指南](../dev/开发环境介绍-业务部署指南.md)  
5. 本地执行 `make inventory`

## 五、未使用的 OSS 高级能力

| 能力 | 控制台状态 | Dev 首期是否需要 |
|------|------------|------------------|
| **接入点（Access Point）** | 无数据 | **否** — 标准 Bucket + RAM 即可 |

## 六、访问控制现状（RAM / ECS）

### 6.1 实机角色与策略

| 项 | 值 |
|----|-----|
| RAM 角色名 | **`Dev-ECS-Role`** |
| 信任主体 | 阿里云 ECS（`ecs.aliyuncs.com`） |
| 自定义策略 | **`DevOssInfraDevFileStorageOnly`** |
| Allow | 仅 `infra-dev-file-storage` 及 `/*` 对象（Get/Put/List 等最小集） |
| Deny | `yzx-file-storage`、`taoyouxiangzu` 及下属对象（`oss:*`） |

### 6.2 ECS 实例 RAM 绑定

| 主机 | 角色 | 是否应绑定 | 台账 `attached` | 实机状态 |
|------|------|------------|-----------------|----------|
| dev-01 / ci-01（同机 yax） | Dev 应用 / CI | **是** | `true` | **已绑定**（2026-06-22） |
| dev-02（112.124.104.223） | Dev Worker | **是**（Bootstrap 后） | `false` | pending_bootstrap |
| hub-01 | 管理面 | **否** | — | 不需要 OSS 写权限 |
| test-01 | Test 预留 | Test 阶段另建角色 | — | 未实施 |

### 6.3 访问方式对照

| 方式 | Dev 是否采用 | 说明 |
|------|--------------|------|
| ECS 实例 RAM 角色 | **是（已验收）** | 应用 SDK 默认凭证链；`make oss-smoke` |
| 长期 AK/SK 写入 `.env` | **否** | 不进 Git |
| Bucket 公共读 | **否** | 私有 + 签名 URL（若前端直传再单独配 CORS/STS） |
| 办公笔记本直连 OSS | 一般不 | 经 Dev API 或临时 STS；与 WG 无直接关系 |

**Ansible 侧**：`bootstrap.yml` 可在 `ram_role_verify=true` 时校验元数据；Put/Get 验收见 **`make oss-smoke`**（`scripts/dev/oss-smoke.sh`）。

## 七、Dev 环境连接参数（应用配置）

以下与 `ansible/inventories/dev/group_vars/all/main.yml` 对齐；**不使用 AccessKey**。

| 变量（建议） | 当前值 | 备注 |
|--------------|--------|------|
| `OSS_BUCKET` / `oss.bucket` | `infra-dev-file-storage` | Dev 专用 Bucket |
| `OSS_PREFIX` / `oss.prefix` | `""`（空） | 桶级隔离；应用键如 `uploads/...`，不再使用 `dev/` 前缀 |
| `OSS_ENDPOINT` | `oss-cn-hangzhou-internal.aliyuncs.com` | ECS 上优先内网 |
| `OSS_REGION` | `oss-cn-hangzhou` | 与 Bucket 地域一致 |
| 状态 | **`operational`** | 2026-06-22 验收 |

**与 RDS 对称的隔离原则**：

- RDS：`app_dev` 账号仅 `app_dev.*`  
- OSS：`Dev-ECS-Role` 仅 `infra-dev-file-storage`；**不得**访问 `yzx-file-storage`
