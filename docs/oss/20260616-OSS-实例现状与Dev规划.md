# OSS 对象存储实例现状与 Dev 规划

> **文档性质**：阿里云 OSS 人工权威台账，供 Dev/Test/Prod 前缀隔离、RAM 角色、应用连接与后续验收引用。  
> **归档日期**：2026-06-16（初版）  
> **前提上下文**：阶段 F（WireGuard）已完成；阶段 G（RDS `app_dev`）已验收；**阶段 H（Dev OSS）尚未验收**；JumpServer 与业务应用尚未开始。  
> **机器可读副本**：`docs/assets/registry.yaml` → `shared_cloud_dependencies.oss`；`ansible/inventories/dev/group_vars/all/main.yml` → `oss`

## 一、文档用途与维护规则

| 原则 | 说明 |
|------|------|
| 控制台变更须回写 | 修改 Bucket 权限、RAM 策略/角色、ECS 实例 RAM 绑定后，同步更新本文档与 `registry.yaml`、`dev-*.yaml` |
| ECS 走实例 RAM 角色 | Dev **禁止**在应用 `.env` 或 Git 中存放长期 OSS AccessKey |
| 按前缀分环境 | 同一 Bucket 内用 `dev/`、`test/`、`prod/` 等前缀隔离；**不默认**为 Dev 新建独立 Bucket |
| 与生产数据共存时须最小权限 | `yzx-file-storage` 已有约 **27 GB** 数据；Dev 角色**仅**允许 `dev/*`，验收须验证无法访问生产路径 |
| 不走 WireGuard | OSS 经阿里云 VPC 内网 Endpoint 或公网 Endpoint 访问，**不经过** `10.200.x.x` WG 隧道 |

**维护流程**：

1. 阿里云 OSS / RAM 控制台确认变更  
2. 更新本文档对应章节  
3. 同步 `docs/assets/registry.yaml`、`docs/assets/dev-01.yaml`（及 dev-02）  
4. 验收通过后更新 `docs/acceptance/`（阶段 H 报告）与 [开发环境业务部署指南](../dev/开发环境介绍-业务部署指南.md)  
5. 本地执行 `make inventory`

## 三、Bucket 详情：`yzx-file-storage`

| 项 | 值 |
|----|-----|
| Bucket 名称 | `yzx-file-storage` |
| 地域 | **华东 1（杭州）** `oss-cn-hangzhou` |
| 存储类型 | **标准存储** |
| 冗余类型 | **本地冗余存储（LRS）** |
| 已用容量 | **约 27.01 GB**（标准型存储量） |
| 低频型存储量 | 0 Byte |
| 当月流量 | 约 **278.81 MB** |
| 当月访问次数 | 约 **2,441** 次 |
| 版本控制 | **未开启** |
| 传输加速 | **已开启** |
| 运行状态 | **使用中**（有存量对象与访问） |

**含义**：

- 该 Bucket **非空桶**，承载历史/生产或混合业务数据。  
- Dev 环境**不应**新建平行 Bucket（除非合规强制），而应通过 **`dev/` 前缀 + RAM 策略** 与生产对象隔离。  
- 传输加速已开：应用在 ECS 上仍应**优先配置内网 Endpoint**；是否走加速域名由 SDK 与业务决定，Dev 验收时需记录实际使用的 Endpoint。

### 3.1 对象路径规划（Dev）

| 环境 | 对象键前缀（示例） | 状态 |
|------|-------------------|------|
| **Dev** | `dev/uploads/`、`dev/static/`、`dev/smoke-test.txt` | **规划**；控制台对象列表待运维确认是否已有 `dev/` 对象 |
| Test（后续） | `test/` | 未实施 |
| Prod（存量） | 非 `dev/` 路径（具体目录待业务台账补充） | **已存在数据**，Dev RAM **不得**授权 |

### 3.2 Endpoint（应用配置参考）

| 类型 | 域名（杭州） | 用途 |
|------|--------------|------|
| **内网**（ECS 推荐） | `oss-cn-hangzhou-internal.aliyuncs.com` | Dev/Test/Prod 应用在 ECS 上默认使用 |
| 公网 | `oss-cn-hangzhou.aliyuncs.com` | 非 ECS 或救急；Dev 应用默认不用 |
| 传输加速 | 控制台「传输加速」对应域名 | 已开启；按业务/SDK 配置，非 Dev 首期必测项 |

Bucket 级访问域名（虚拟主机风格示例）：

- 内网：`yzx-file-storage.oss-cn-hangzhou-internal.aliyuncs.com`  
- 公网：`yzx-file-storage.oss-cn-hangzhou.aliyuncs.com`

## 五、访问控制现状（RAM / ECS）

### 5.1 规划角色

| 项 | 值 |
|----|-----|
| RAM 角色名 | **`Dev-ECS-Role`** |
| 信任主体 | 阿里云 ECS（`ecs.aliyuncs.com`） |
| 策略意图 | 仅允许对 `yzx-file-storage` 的 **`dev/*`** 对象读写；**禁止** `prod/*` 及 Bucket 内其他生产路径 |
| 第二 Bucket | 若创建 `taoyouxiangzu`，策略中单独增加 `dev/*` Resource（当前 Bucket 不存在，**策略可暂不包含**） |

### 5.2 ECS 实例 RAM 绑定

| 主机 | 角色 | 是否应绑定 | 台账 `attached` | 实机状态 |
|------|------|------------|-----------------|----------|
| dev-01 / ci-01（同机 yax） | Dev 应用 / CI | **是** | `false` | **待绑定** |
| dev-02（112.124.104.223） | Dev Worker | **是**（Bootstrap 后） | — | pending_bootstrap |
| hub-01 | 管理面 | **否** | — | 不需要 OSS 写权限 |
| test-01 | Test 预留 | Test 阶段另建角色 | — | 未实施 |

### 5.3 访问方式对照

| 方式 | Dev 是否采用 | 说明 |
|------|--------------|------|
| ECS 实例 RAM 角色 | **是（目标态）** | 应用 SDK 默认凭证链 |
| 长期 AK/SK 写入 `.env` | **否** | 不进 Git |
| Bucket 公共读 | **否** | 私有 + 签名 URL（若前端直传再单独配 CORS/STS） |
| 办公笔记本直连 OSS | 一般不 | 经 Dev API 或临时 STS；与 WG 无直接关系 |

**Ansible 侧**：`bootstrap.yml` 可在 `ram_role_verify=true` 时校验元数据服务是否返回 `Dev-ECS-Role`；**绑定操作须在阿里云控制台完成**，Ansible 不能代替创建 RAM 策略。

## 七、与 ECS / WireGuard / RDS 的关系

```
dev-01 (172.21.226.38) ──RAM 角色──→ OSS 内网 Endpoint ──→ yzx-file-storage / dev/*
         ↑
    不经过 wg0 (10.200.x.x)

dev-01 ──VPC 内网──→ RDS app_dev     （阶段 G 已验收，与 OSS 并行、无依赖）

hub-01 / JumpServer（规划）         → 通常不需要 OSS 权限
办公笔记本 + WG                     → 不能替代 ECS 上的 RAM 角色访问 OSS
```

| 组件 | 是否依赖 OSS | 说明 |
|------|--------------|------|
| WireGuard | 否 | 阶段 F 已完成 |
| RDS `app_dev` | 否 | 阶段 G 已完成；应用可同时使用 DB + OSS |
| JumpServer（规划） | 否 | 堡垒机不依赖对象存储 |
| Dev 应用（规划） | **视业务** | 若上传文件/静态资源，需要本节 RAM + 前缀验收 |
| Bootstrap `ram_role_verify` | 部分 | 可验证元数据与角色名，**不能替代** PutObject/GetObject smoke |

## 九、推荐实施步骤（阶段 H）

**OSS Dev 可用**尚未完成；建议顺序（**先于 JumpServer**）：

1. **H0 现状确认**：在 `yzx-file-storage` 文件列表中确认是否已有 `dev/` 前缀对象；确认 Bucket 权限/公共访问设置。  
2. **H1 RAM**：创建/复核 `Dev-ECS-Role` 及自定义策略（仅 `yzx-file-storage` + `dev/*`）。  
3. **H2 Bucket 加固**：阻止公共访问；确认加密与日志（按合规）。  
4. **H3 绑定 ECS**：dev-01（yax）绑定 `Dev-ECS-Role`；dev-02 就绪后补绑。  
5. **H4 实机验收**（在 dev-01/ci-01 上）：元数据角色名 → Put/Get `dev/smoke-test-*` → 访问非 `dev/` 路径被拒绝。  
6. **H5 回写台账**：`dev-01.yaml`、`registry.yaml`、本文档 §十清单；编写 `docs/acceptance/20260616-阶段H-Dev-OSS验收.md`。  
7. **H6 业务指南**：更新 [开发环境介绍-业务部署指南](../dev/开发环境介绍-业务部署指南.md) 第六节与路线图。

**第二 Bucket `taoyouxiangzu`**：与业务确认前，步骤 1–6 可仅针对 `yzx-file-storage`。
