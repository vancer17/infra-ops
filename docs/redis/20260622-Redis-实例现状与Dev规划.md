> **前提上下文**：阶段 F（WireGuard）、阶段 G（RDS `app_dev`）、阶段 H（OSS）已完成；**阶段 I（Dev 云 Redis）实机已验收**。  
> **机器可读副本**：`docs/assets/registry.yaml` → `shared_cloud_dependencies.redis`；`ansible/inventories/dev/group_vars/all/network.yml` → `redis` / `redis_whitelist`

## 一、文档用途与维护规则

| 原则 | 说明 |
|------|------|
| 控制台变更须回写 | 修改白名单、账号、连接地址后，同步本文档、`registry.yaml`、`network.yml` |
| ECS 走内网、不走 WG | Redis 白名单填 **ECS VPC 私网 IP**，不填 WireGuard `10.200.x.x` |
| 应用连内网域名 | Dev ECS 使用 **专有网络（内网）连接地址** |
| 与 MySQL 区分 | Redis **无** `CREATE DATABASE`；Dev 采用 **独立账号 + DB 0 + 键前缀** |
| 密码不进 Git | 账号密码放 GitHub Secret `REDIS_PASSWORD` 或 ansible-vault |

**维护流程**：

1. 阿里云 Redis 控制台确认变更  
2. 更新本文档对应章节  
3. 同步 `docs/assets/registry.yaml`、`ansible/inventories/dev/group_vars/all/network.yml`  
4. 本地执行 `make inventory`

## 六、与 ECS / WireGuard 的关系

```
dev-01 (172.21.226.38) ──VPC 内网 eth0──→ Redis 内网 endpoint:6379
         ↑
    不经过 wg0 (10.200.x.x)

ci-01 与 dev-01 同机 → 验收在 dev-01/ci-01 上执行 nc / redis-cli 即可
Hub / JumpServer     → 通常不需要 Redis 白名单
办公笔记本 + WG      → 不能替代 ECS 连 Redis
```

| 组件 | 是否依赖 Redis | 说明 |
|------|----------------|------|
| WireGuard | 否 | 已完成 |
| JumpServer | 否 | 人工运维可选登记数据源 |
| Dev 应用 | **是** | `petintelli-backend` 配置 `REDIS_*` |
| dev-02 Docker Redis | **否** | 已由云 Redis 替代 |
